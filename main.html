TCGA Toolbox

TCGA Toolbox provides a scripting environment to access, process, and analyze data from TCGA in the browser. It differs from other web apps in that the main interaction happens in the JavaScript Console.

This is a work in progress. The code is hosted on GitHub.

Modules

TCGA Toolbox can be extended by loading remote scripts – either by specifying the URL or by selecting one from our trusted authors.

    Documentation

    Everything in the TCGA Toolbox is scoped under the global TCGA namespace.

    Unless stated otherwise, the first argument of a callback function is reserved for an error object, a useful convention to allow error catching in asynchronous programming. If the error object is null no error occurred.

    For end users

    Load scripts

    TCGA.loadScript(url, callback) Loads a script using an HTML script element.
    url is a string representing a valid URL, callback a function with two parameters, the error object and a string array of loaded scripts.
    TCGA.loadScript([url, url, ...], callback) Loads multiple scripts in sequence using HTML script elements.
    [url, url, ...] is a string array of valid URLs, callback a function with two parameters, the error object and a string array of loaded scripts.
    TCGA.modules(callback) Returns a list of loaded modules.
    callback is a function with two parameters, the error object and a string array of loaded modules.

    Find data

    TCGA.find(query, callback) Queries the TCGA RDF file index.
    query is a string representing a SPARQL query, callback a function with two parameters, the error object and an object representing the SPARQL query results in the SPARQL 1.1 Query Results JSON Format.

    Get data

    TCGA.get(url, callback) Downloads the contents of a file from TCGA's open-access HTTP directory or any CORS-enabled service.
    url is a string representing a valid URL, callback a function with two parameters, the error object and a string representing the contents of the requested file.
    TCGA.getJSON(url, callback) Same as TCGA.get, but parses the result to JSON.
    url is a string representing a valid URL, callback a function with two parameters, the error object and an object representing the contents of the requested file as JSON.
    TCGA.getXML(url, callback) Same as TCGA.get, but parses the result to XML.
    url is a string representing a valid URL, callback a function with two parameters, the error object and an object representing the contents of the requested file as XML.
    TCGA.getRange(url, startByte, endByte, callback) Same as TCGA.get, but downloads only a subset of the file within the specified byte range.
    url is a string representing a valid URL, startByte and endByte the integer range of bytes to download, and callback a function with two parameters, the error object and a string representing the partial contents of the requested file.
    TCGA.getArchive(url, callback) Downloads an archive from the TCGA open-access HTTP directory or CORS-enabled services.
    url is a string representing a valid URL, callback a function with two parameters, the error object and an object representing the contents of the requested archive.

    Store data temporarily

    TCGA.data We are trying to establish a convention that modules should put data that they want to share with other modules in a place where they can find them. TCGA.data is the simplest solution for that problem: it is a regular JavaScript object in the TCGA namespace. Be aware of the consequences when working with it: you will work with references rather than copies unless you copy the data yourself, e.g. JSON.parse(JSON.stringify(TCGA.data["data-from-some-other-module"])).

    Store data permanently (don't delete your browsing history)

    TCGA.store.set(key, value, callback) Adds a key-value pair to the store.
    key is a string representing the key, value any type representing the value, and callback a function with one parameter, the error object.
    TCGA.store.get(key, callback) Retrieves a key-value pair from the store.
    key is a string representing the key, callback a function with two parameters, the error object and any type representing the value.
    TCGA.store.del(key, callback) Removes a key-value pair from the store.
    key is a string representing the key, callback a function with one parameter, the error object.
    TCGA.store.exists(key, callback) Checks if a key-value pair with the specified key exists in the store.
    key is a string representing the key, callback a function with two parameters, the error object and a boolean indicating if it is in the store or not.
    TCGA.store.keys(callback) Returns a list of all keys in the stores.
    callback is a function with two parameters, the error object and a string array of keys.
    TCGA.store.clear(callback) Removes all key-value pairs from the store.
    callback is a function with one parameter, the error object.

    For script authors

    Register tab in main menu

    TCGA.registerTab(options) Registers a tab in the menu bar of the TCGA Toolbox.
    options is an object with the following properties: id is a string representing the ID of the tab (we recommend to name it after your module), title is a string representing the title of your module, and switchTab is a boolean indicating whether the TCGA Toolbox should immediately switch to your tab after it was created.

    Display notifications

    TCGA.toast.info(message, duration) Displays an info notification with the specified string message for the specified duration in milliseconds.
    TCGA.toast.success(message, duration) Displays a success notification with the specified string message for the specified duration in milliseconds.
    TCGA.toast.error(message, duration) Displays an error notification with the specified string message for the specified duration in milliseconds.

    Get version information

    TCGA.version() Returns the current version of the TCGA Toolbox.
